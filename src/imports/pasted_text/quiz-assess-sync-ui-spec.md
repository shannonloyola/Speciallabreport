Create a modern light-mode web app UI called “QuizAssess Sync: Concept Alignment and Coverage Analyzer”.

This is a student-focused educational analysis system that compares quiz questions with PowerPoint presentation slides to determine:
- which slides best match each quiz question,
- what concept/topic each question is testing,
- whether each question is fully covered, partially covered, or not covered by the presentation,
- whether there is a mismatch between what the quiz demands and what the slides actually teach,
- and what recommendations can help improve either the quiz or the presentation.

The UI must look clean, modern, smart, and easy to navigate. It should feel like an intelligent academic tool, not a cluttered dashboard. The design should prioritize clarity, simplicity, and smooth user flow. Use a polished student-friendly interface with a modern but professional academic feel.

Design direction:
- light mode only
- modern web app
- clean and minimal layout
- student-friendly but still professional
- strong visual hierarchy
- simple navigation
- card-based sections
- lots of breathing space
- readable typography
- no clutter
- no overly technical or crowded screens
- not corporate finance style
- not too childish

Use this visual style:
- white or soft off-white background
- dark gray text
- muted blue as primary accent
- soft green for fully covered
- soft amber or yellow for partially covered
- soft red for not covered
- light gray borders and dividers
- rounded cards
- soft shadows
- clean sans-serif typography
- medium-to-large spacing
- consistent icons and components

The app should include these pages:

1. Home / Welcome Page
Purpose: clearly explain what the system does in a simple way for students.

Include:
- app name: QuizAssess Sync
- subtitle: “Concept Alignment and Coverage Analyzer”
- a short explanation that the app checks whether quiz questions are aligned with lesson slides using semantic analysis, concept matching, coverage classification, and recommendation generation
- primary CTA button: Start Analysis
- secondary CTA button: View Demo
- a simple 3-step process section:
  Upload Quiz and Slides
  Analyze Alignment
  Review Insights and Recommendations
- a clean feature preview section with 4 cards:
  Semantic Matching
  Concept Detection
  Coverage Analysis
  Smart Recommendations
- a hero illustration or abstract educational AI visual
- modern student-focused layout

2. Upload and Setup Page
Purpose: allow students to upload files and configure the analysis.

Include:
- drag-and-drop upload area for Quiz File
- drag-and-drop upload area for PowerPoint / Slides File
- visible accepted file formats
- uploaded file preview cards
- a right-side or top-side configuration panel with:
  Analysis Mode
  Matching Depth
  Coverage Sensitivity
  Enable Cognitive-Level Check toggle
- a prominent Run Analysis button
- simple progress state for uploaded files
- very clean, guided layout with short helper text
- avoid making this page look complicated

3. Main Dashboard Page
Purpose: show a clear overview of results.

Include:
- top summary cards:
  Total Questions
  Fully Covered
  Partially Covered
  Not Covered
  Overall Alignment Score
- a clean chart for coverage distribution
- a concept/topic overview section showing which concepts appear most in the quiz and in the slides
- a cognitive mismatch summary card
- a recommendation summary card with 2 to 4 written recommendations
- a clean “Question Analysis Table” with these columns:
  Question No.
  Detected Concept
  Best Matching Slide
  Coverage Status
  Confidence
  View Details
- filters for concept, status, or confidence
- search bar for question lookup
- use tabs or segmented controls if needed to keep the page tidy
- do not overcrowd the dashboard

4. Question Detail Page
Purpose: provide deep analysis for one selected question.

Include:
- full quiz question text at the top
- detected concept/topic tags
- best matching slide card
- second-best supporting slide card
- coverage status badge:
  Fully Covered
  Partially Covered
  Not Covered
- a plain-language explanation of why the question received that classification
- supporting evidence section with matched slide excerpts
- a comparison section such as:
  Quiz Requires
  Slides Support
- cognitive-level comparison such as:
  Understand / Apply / Analyze
- recommendation box with examples like:
  “Review Slide 8 before answering this question”
  “This question expects deeper explanation than the current slides provide”
  “Add a worked example to support this concept”
- confidence indicator
- student-friendly wording and readable layout

5. Recommendations Page
Purpose: turn results into actionable guidance.

Include:
- Key Findings section
- Alignment Issues section
- Improvement Recommendations section
- Overrepresented Topics section
- Underrepresented Topics section
- Cognitive-Level Mismatch Insights section
- downloadable report buttons:
  Export PDF
  Export Summary
- highlight written interpretation more than raw numbers

6. Insights / Visualization Page
Purpose: show elegant supporting analytics without being too technical.

Include:
- concept coverage heatmap
- question-to-slide mapping visualization
- topic balance chart
- cognitive-level comparison chart
- simple legends and readable labels
- keep visualizations polished and minimal

Navigation:

- Home
